3. ОПРЕДЕЛЕНИЯ И ТИПЫ ЛИЦЕНЗИЙ.

3.1. Программная лицензия. Программная лицензия позволяет вам установить один экземпляр Программного обеспечения в среде с одним или несколькими серверами и разрешает пользователям, имеющим Лицензию зарегистрированного пользователя (ЛЗП) на данное Программное обеспечение, получать доступ к Программному обеспечению и использовать его. Каждая Программная лицензия предназначена для одного развертывания и не может использоваться совместно для других развертываний.

3.2. Лицензия зарегистрированного пользователя (ЛЗП). Если лицензия на Программное обеспечение предоставляется Зарегистрированному пользователю, то каждый такой пользователь должен быть отдельно указан в качестве единственного обладателя ЛЗП. Совместное использование ЛЗП несколькими лицами прямо запрещено. Кроме того, ЛЗП не подлежит передаче от одного лица другому, пока исходный конечный пользователь не перестанет нуждаться в Программном обеспечении и более не будет иметь разрешения на обращение к Программному обеспечению.

3.3. Лицензия на одновременный доступ ("ЛОД"). Когда Программное обеспечение лицензируется на основе одновременного доступа, суммарное количество конечных пользователей, обращающихся к Программному обеспечению в любой момент времени, не может превышать числа полученных ЛОД. ЛОД предназначена(ы) для определенного Развертывания и не могут использоваться совместно для других Развертываний. При использовании Лицензий на одновременный доступ нельзя использовать программу или систему для кэширования или создания очереди запросов на создание отчета. 

3.4. Лицензия на основе Процессора или ЦП Если Программное обеспечение лицензируется на основе процессора или ЦП, то суммарное количество центральных процессоров ("Процессоров"), на которых запускаются какие-либо компоненты Программного обеспечения (кроме следующих: Web Connector, SDK, Report Publishing Wizard и средства просмотра отчетов), не может превышать числа лицензированных процессоров или ЦП. В отношении многоядерного процессора с количеством процессорных ядер Nприменяется следующий метод расчета: первое процессорное ядро в каждом физическом ЦП считается за один процессор или ЦП, каждое инкрементное процессорное ядро в физическом ЦП считается за 0,5 процессора или ЦП. 

3.5. Лицензия на сервер. Когда Программное обеспечение лицензируется на основе сервера то, его можно загружать только на один компьютер, содержащий не более четырех (4) процессоров или ЦП. В отношении многоядерного процессора с количеством процессорных ядер Nприменяется следующий метод расчета: первое процессорное ядро в каждом физическом ЦП считается за один процессор или ЦП, каждое последующее процессорное ядро в физическом ЦП считается за 0,5 процессора. 

3.6. Лицензия на основе подписки. Если Программное обеспечение лицензируется на основе подписки, то вам предоставляется неисключительная лицензия, не подлежащая передаче другому лицу, которая разрешает использовать Программное обеспечение в течение двенадцати месяцев и должна ежегодно (или по прохождении другого срока, согласованного между сторонами) обновляться у лица, предоставившего лицензию. 

3.7. Ограниченная лицензия. Если Программное обеспечение получено в составе, в сочетании или для использования совместно с программным продуктом стороннего разработчика ("Приложение OEM"), то это означает, что получена ограниченная лицензия. Каждую лицензированную копию ПО можно использовать только совместно с Приложением OEM, в составе которого оно предоставлено. Доступ к данным, которые не созданы или не используются с помощью Приложения OEM, является нарушением этой лицензии. Если в Приложении OEM требуется использование подгруппы данных или хранилища данных, то Программное обеспечение можно использовать с подгруппой данных или хранилищем данных только для доступа к данным, созданным или обработанным с помощью этого Приложения OEM. Если совместно с Приложениями планирования вы приобрели Сrystal Xcelsius, то в отношении данного продукта вы приобретаете Ограниченную лицензию. Crystal Xcelsius можно использовать только в сочетании с Приложениями планирования, и доступ к данным, которые были созданы или используются не в Приложениях планирования, является нарушением лицензионного соглашения. Ограниченную лицензию нельзя объединять с неограниченными лицензиями в рамках одного Развертывания.
